# Sidebar Navigation Guide

## How Streamlit Sidebar Navigation Works

In Streamlit multi-page apps, the sidebar navigation is **automatically generated** by Streamlit based on the files in the `pages/` directory.

## Automatic Navigation

Streamlit automatically creates a navigation menu in the sidebar when you have a `pages/` directory with Python files. The navigation appears at the **top of the sidebar**.

4. **Navigation**: Clicking a page name navigates to that page

## Customizing Navigation

### Option 1: Rename/Reorder Pages
Change the number prefix in the filename:
- `1_ğŸ“Š_Dashboard.py` (appears first)
- `2_âš™ï¸_Assumptions.py` (appears second)
- etc.

### Option 2: Hide Pages
Remove files from `pages/` directory or rename to not start with a number.

### Option 3: Add Custom Navigation
You can add custom navigation elements in the sidebar using `st.sidebar`:

```python
# In any page file
with st.sidebar:
    st.markdown("### Custom Navigation")
    if st.button("Go to Dashboard"):
        st.switch_page("pages/1_ğŸ“Š_Dashboard.py")
```

## Finding the Navigation

1. **Run the app**: `streamlit run app.py`
2. **Look at the sidebar** (left side of the screen)
3. **Top of sidebar**: You'll see the page navigation menu
4. **Below that**: Page-specific content (like inputs on Assumptions page)

## Troubleshooting

**Q: I don't see the navigation menu**
- Make sure you have files in the `pages/` directory
- Files must start with a number (e.g., `1_`, `2_`)
- Restart Streamlit: `streamlit run app.py`

**Q: Pages are in wrong order**
- Check the number prefix in filenames
- `1_` comes before `2_`, etc.

**Q: Want to customize the menu**
- Streamlit's auto-navigation is limited
- You can add custom buttons using `st.sidebar.button()` and `st.switch_page()`
- Or use `st.navigation()` for more control (Streamlit 1.28+)
